## Prerequisites

- **macOS 12+** or Linux
- **Node.js 18+** (check: `node --version`)
- **Python 3.9+** (check: `python3 --version`)
- **Claude Desktop** (latest version) — [Download](https://claude.ai/download)
- **Git** (check: `git --version`)
- Text editor (VS Code recommended, or any IDE)

## Step 1: Clone the Repository

```bash
git clone https://github.com/puzzled-mushroom911/claude-code-real-estate-framework.git
cd claude-code-real-estate-framework
```

Verify structure:
```bash
ls -la
# Should show: .claude/, knowledge_bases/, mcp_server/, tools/, config/, scripts/, README.md
```

## Step 2: Install Node.js Dependencies

```bash
npm install
```

This installs MCP server dependencies and tooling. Verify:
```bash
npm list | head -20
```

## Step 3: Install Python Virtual Environment & Dependencies

```bash
# Create virtual environment in project root
python3 -m venv venv

# Activate virtual environment
source venv/bin/activate  # macOS/Linux
# OR on Windows:
# venv\Scripts\activate

# Install Python dependencies
pip install -r requirements.txt
```

Verify activation (your prompt should show `(venv)`):
```bash
which python3
# Should output: /path/to/project/venv/bin/python3
```

## Step 4: Configure Environment Variables

Create a `.env` file in the project root:

```bash
cp .env.example .env
```

Edit `.env` with your configuration:

```env
# MCP Server
MCP_SERVER_PORT=3000
MCP_SERVER_HOST=localhost

# Knowledge Bases (local paths)
KNOWLEDGE_BASE_PATH=./knowledge_bases
VECTOR_DB_PATH=./knowledge_bases/vectors

# Optional: API Keys (if using external services)
# OPENAI_API_KEY=your_key_here
# MLS_API_KEY=your_key_here
```

**Important:** Never commit `.env` to version control. It's listed in `.gitignore`.

## Step 5: Start the MCP Server

In a terminal, keep running:

```bash
npm run mcp:start
```

Expected output:
```
MCP Server listening on localhost:3000
Knowledge bases indexed: 4
Vector embeddings loaded
Ready for Claude connections
```

**Keep this terminal open** while using Claude Desktop. The server must be running for MCP tools to work.

## Step 6: Register MCP with Claude Desktop

Open Claude Desktop settings:

1. Launch **Claude Desktop**
2. Click your profile icon (top right) → **Settings** (or **Preferences**)
3. Go to **Developer** tab
4. Click **"Add MCP server"** or **"Configure MCP"**

Fill in the form:

| Field | Value |
|-------|-------|
| **Name** | `realtor-os-local` (or your preferred name) |
| **Type** | `stdio` |
| **Command** | `node` |
| **Args** | `["/path/to/project/mcp_server/index.js"]` |

Replace `/path/to/project/` with actual path. Get it via:
```bash
pwd  # Copy this output
```

Example (if project is in `/Users/you/dev/claude-code-real-estate-framework`):

```json
{
  "name": "realtor-os-local",
  "type": "stdio",
  "command": "node",
  "args": ["/Users/you/dev/claude-code-real-estate-framework/mcp_server/index.js"]
}
```

5. Click **"Save"** or **"Confirm"**

## Step 7: Verify MCP Connection

In a new Claude Desktop chat:

1. Click the **Tools** icon (bottom of chat input, or `@` mention)
2. You should see MCP tools listed: `mcp_knowledge_bases`, `mcp_search`, `mcp_file_system`, etc.
3. Type a test query:

```
@mcp_search What is a curiosity gap?
```

Expected response: Claude searches the wiki and returns relevant results.

## Troubleshooting

### MCP Server Won't Start

**Error:** `Cannot find module 'mcp-sdk'`

**Fix:**
```bash
npm install
npm run build  # if build script exists
```

**Error:** `Port 3000 already in use`

**Fix:** Change port in `.env`:
```env
MCP_SERVER_PORT=3001
```

Then restart server.

### Python Venv Issues

**Error:** `command not found: python3`

**Fix:** Install Python 3.9+ via Homebrew (macOS):
```bash
brew install python@3.9
```

**Error:** `No module named 'X'`

**Fix:** Ensure venv is activated and requirements installed:
```bash
source venv/bin/activate
pip install -r requirements.txt
```

### Claude Desktop Doesn't See Tools

**Error:** "MCP tools not available" or tools menu is empty

**Fix:**
1. Verify MCP server is running (check terminal)
2. Restart Claude Desktop completely (quit and reopen)
3. Double-check path in MCP config matches actual project location:
   ```bash
   ls /path/from/config/mcp_server/index.js
   ```
