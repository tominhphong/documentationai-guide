# Framework Overview

> Source: `README.md`

The Claude Code Real Estate Framework is an AI assistant architecture designed for real estate agents who use YouTube as their primary lead source. It combines specialized agents, MCP tool integration, Python data processing, and pre-loaded marketing knowledge to automate the content-to-client pipeline: from video creation through content repurposing, lead nurturing, CRM operations, and client communication.

Everything runs locally on your Mac. No cloud processing, no monthly fees beyond Claude Code itself. Your data stays on your computer.

## What's Included

### 9 Specialized Agents

Each agent handles a specific phase of your workflow:

- **youtube-video-pipeline** — Full lifecycle: research competitor coverage, create filming outlines, convert transcripts to teleprompter scripts, execute post-production (blog, emails, social clips, YouTube metadata)
- **youtube-to-blog** — Convert video transcripts into WordPress-ready SEO blog posts with embedded video, H2/H3 structure, FAQ sections, and SEO reports
- **email-writer** — Generate lead nurture sequences using Russell Brunson's Daily Seinfeld framework (story-hook-value-invite pattern, 200 words max, plain text)
- **teleprompter** — Convert video outlines into delivery scripts with timing markers, pacing notes (`[SLOW]`, `[PAUSE 2s]`), emphasis cues, and reading-speed calibration
- **content-repurposer** — Take any content source (blog, video, email) and produce multiple formats while maintaining your voice
- **voice-processor** — Ingest iPhone voice note transcripts, extract action items and instructions, organize output
- **crm-agent** — Search/create/update contacts, send SMS/email, manage pipeline stages, handle escalations
- **customer-response** — Draft style-matched client replies by querying conversation history and knowledge base, with inquiry classification
- **relocation-guide-generator** — Create personalized relocation guides from your content using questionnaire-driven approach, output structured JSON for professional PDF production

## Design Principles

**AS-IS documentation** — This framework documents how the system actually works, not how it could work.

**Paraphrase, not copy** — Guides pull knowledge from the framework but use their own language and examples.

**ClaudeKit-first** — All integrations and tool connections flow through Model Context Protocol (MCP) and Claude Code, not through external web UIs.

**File ownership discipline** — Each agent has an `.md` file in `.claude/agents/`, each rule lives in `.claude/rules/`, each skill in `.claude/skills/`. No monolithic instruction files.

**Frontmatter discipline** — Every agent and skill includes YAML metadata: task type, preconditions, output format, and success criteria.
