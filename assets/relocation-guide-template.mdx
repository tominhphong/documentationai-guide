# Relocation Guide Template

> Source: `templates/relocation_guide_template.json`

The relocation guide template defines the structural schema for a 40-page PDF lead magnet. The relocation-guide-generator agent populates each section with city-specific content extracted from RAG data (agent YouTube videos and transcripts). Template variables (`{{AGENT_NAME}}`, `{{CITY}}`, etc.) are substituted from the agent profile.

## Purpose

This template ensures consistent structure across all agent relocation guides while allowing dynamic content population. The schema maps:
- Page ranges (e.g., "neighborhoods" spans pages 11–16)
- Layout type for each section (e.g., `neighborhood_deep_dive`)
- Data fields to extract and fill from RAG
- Branding variables (colors, agent contact info)

The generator reads this schema, queries RAG for relevant content, and produces a PDF with consistent design and information hierarchy.

Cross-reference: `/agent-instructions/relocation-guide-generator` describes the agent workflow that fills this template.

## Top-Level Structure

```json
{
  "metadata": {...},
  "branding": {...},
  "pages": {...}
}
```

### Metadata

```json
"metadata": {
  "version": "1.0",
  "page_count": 40,
  "description": "Relocation guide template for real estate agents. Content populated from YouTube RAG data."
}
```

| Field | Type | Purpose |
|-------|------|---------|
| `version` | string | Schema version (for compatibility tracking) |
| `page_count` | integer | Expected total pages (40) |
| `description` | string | Human-readable guide purpose |

### Branding

```json
"branding": {
  "agent_name": "{{AGENT_NAME}}",
  "brokerage": "{{AGENT_BROKERAGE}}",
  "phone": "{{AGENT_PHONE}}",
  "email": "{{AGENT_EMAIL}}",
  "website": "{{AGENT_WEBSITE}}",
  "primary_color": "#003366",
  "accent_color": "#C9913B",
  "background_color": "#F5F1E8"
}
```

All `{{VAR}}` placeholders are replaced from `/assets/agent-profile-config` during generation. Color codes define the PDF visual theme.

## Pages Map

The `pages` object contains a key per section (e.g., `"cover"`, `"neighborhoods"`, `"resources"`). Each section specifies:
- **`page`** or **`pages`** – Page number(s) or range
- **`layout`** – Visual layout template (CSS/design class)
- **`content`** – Data fields to populate

### Cover (Page 1)

```json
"cover": {
  "page": 1,
  "layout": "cover",
  "content": {
    "title": "Your Complete Guide to Relocating to {{CITY}}",
    "subtitle": "Everything you need to know before making the move",
    "agent_line": "By {{AGENT_NAME}} | {{AGENT_BROKERAGE}}"
  }
}
```

**Purpose:** Title page. Variables substituted at generation time.

}

**Purpose:** Auto-populated TOC. No manual content needed.

### Area Overviews (Pages 7–10)

```json
"area_overviews": {
  "pages": [7, 8, 9, 10],
  "layout": "city_profile",
  "content": {
    "cities": [
      {
        "name": "[CITY_1]",
        "description": "[DATA_NEEDED]",
        "lifestyle": "[DATA_NEEDED]",
        "attractions": "[DATA_NEEDED]",
        "vibe": "[DATA_NEEDED]",
        "development_stage": "[DATA_NEEDED]",
        "pros": "[DATA_NEEDED]",
        "cons": "[DATA_NEEDED]",
        "commute_times": "[DATA_NEEDED]"
      },
      {
        "name": "[CITY_2]",
        ...similar structure...
      }
    ]
  }
}
```

**Purpose:** Multi-city comparison. RAG generator populates each city's profile from agent video content.

**Fields per city:**
- `name` – City name
- `description` – Overview and character
- `lifestyle` – Living feel and culture
- `attractions` – Key points of interest
- `vibe` – Tone/personality (e.g., "suburban", "urban", "family-friendly")
- `development_stage` – Maturity (e.g., "rapidly growing", "established", "emerging")
- `pros` – Positive attributes
- `cons` – Realistic drawbacks
- `commute_times` – Travel times to key hubs (in minutes)

### Neighborhoods (Pages 11–16)

```json
"neighborhoods": {
  "pages": [11, 12, 13, 14, 15, 16],
  "layout": "neighborhood_deep_dive",
  "content": {
    "neighborhoods": [
      {
        "name": "[NEIGHBORHOOD_1]",
        "city": "[CITY]",
        "vibe": "[DATA_NEEDED]",
        "housing_types": "[DATA_NEEDED]",
        "price_range": "[DATA_NEEDED]",
        "schools": "[DATA_NEEDED]",
        "location_access": "[DATA_NEEDED]",
        "who_lives_here": "[DATA_NEEDED]",
        "agent_take": "[DATA_NEEDED]"
      }
    ]
  }
}
```

**Purpose:** Detailed neighborhood profiles (typically 2–3 neighborhoods × 2 pages each).

**Fields per neighborhood:**
- `name` – Neighborhood or district name
- `city` – Parent city
- `vibe` – Character and feel
- `housing_types` – Mix (e.g., "single-family, townhomes, condos")
- `price_range` – Median home prices or price bracket
- `schools` – School options and rankings
- `location_access` – Distance to shopping, highways, downtown
- `who_lives_here` – Demographics and lifestyle fit
- `agent_take` – Agent's personal opinion and insider perspective
