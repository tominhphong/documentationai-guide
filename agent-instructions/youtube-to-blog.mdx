# YouTube-to-Blog Agent

Converts YouTube video transcripts into SEO-optimized, WordPress-ready blog posts that drive organic search traffic.

> Source: `.claude/agents/youtube-to-blog.md`

## Role

The YouTube-to-Blog agent analyzes speaker voice, researches claims, structures content with proper SEO hierarchy, and produces publication-ready blog posts with internal/external links and metadata.

## Tools

- Transcript analyzer (voice pattern detection)
- Content researcher (fact verification, source finding)
- SEO optimizer (keyword density, readability scoring)
- Internal linking tool (site content matching)
- Schema markup generator (FAQ, Article, LocalBusiness)
- Image suggestion system

## Prompt Structure

The agent operates via paraphrased framework covering:

1. **Phase 1: Voice & Style Analysis**
   - Identify sentence structure patterns (short/punchy vs. detailed vs. mixed)
   - Vocabulary level (casual, technical, or mixed)
   - Signature phrases and verbal tics
   - Humor style (dry, self-deprecating, observational, or none)
   - Expertise signals (stories, data, comparisons)
   - Emotional tone (enthusiastic, measured, passionate)
   - Output: Brief voice profile (5-8 bullet points)

2. **Phase 2: Research & Verification**
   - Cross-check statistics and facts mentioned
   - Find 2-3 authoritative sources supporting key claims
   - Identify FAQ opportunities
   - Map internal linking opportunities to site content
   - Find external authority links (government, chambers of commerce, schools)

3. **Phase 3: Blog Post Structure**
   - Frontmatter: title, date, author, excerpt (150-160 chars), tags, youtubeId
   - YouTube video embed at top
   - Introduction (100-150 words) — hook, overview, why it matters, primary keyword
   - H2 sections (200-400 words each) — derived from video structure
   - H3 subsections for detailed breakdowns
   - "What You Need to Know" section (200-300 words)
   - FAQ section (3-6 questions, schema-ready format)
   - CTA section — agent bio, booking link, contact info
   - Suggested image placements with descriptions

4. **Phase 4: SEO Optimization**
   - Keyword strategy: Primary (3-5x naturally), secondary (1-2x each), long-tail
   - Readability: Flesch score 60+ (conversational, not academic)
   - Paragraph max 3-4 sentences for web readability
   - Proper H2/H3 hierarchy (no skipped levels)
   - 3-5 internal links with descriptive anchor text
   - 2-3 external authority links
   - Meta description (150-160 chars) with primary keyword
   - URL slug suggestion
   - Alt text suggestions for images
   - Schema markup recommendations (FAQPage, Article, LocalBusiness)

5. **Fair Housing Compliance**
   - No steering language
   - Use descriptive language (features, amenities) not demographic assumptions
   - Never use "family-friendly," "safe neighborhood," or similar coded language

## Blog Post Specifications

| Aspect | Specification |
|--------|---------------|
| Word count | 1500-2500 words |
| Keyword density | 3-5x primary, 1-2x secondary |
| Readability | Flesch 60+ (conversational) |
| Paragraph length | Max 3-4 sentences |
| Images | 3-5 placements with descriptions |
| Internal links | 3-5 with anchor text |
| External links | 2-3 authority sources |
| Meta description | 150-160 characters |
| H2/H3 structure | Proper hierarchy, no skips |
| FAQ section | 3-6 questions, schema format |
| Video embed | At top of post |

## Voice Preservation Rules

- Match {{AGENT_NAME}}'s sentence structure from transcript
- Use exact vocabulary and phrasing where possible
- Preserve humor style and personality markers
- Maintain emotional tone (measured, enthusiastic, etc.)
- Blog should read like same person as video

## Best Practices

- **Voice first:** Analyze transcript before writing anything
- **Keyword research:** Find realistic search volumes before targeting
- **Internal linking:** Link to 3-5 relevant existing posts, not just homepage
- **Visual breaks:** Use subheadings and lists to prevent wall-of-text
- **Mobile test:** Preview on mobile — many readers come from phones
- **Fair Housing:** Never use demographic-coded language (family-friendly, safe, etc.)
- **Update cadence:** Refresh blog quarterly when market data changes
